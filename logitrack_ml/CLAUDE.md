# CLAUDE.md

This file provides guidance to Claude Code when working with the LogiTrack ML Priority Prediction service.

## What is this?

A Python microservice that predicts shipment priority (alta / media / baja) based on 6 shipment factors. It uses a RandomForest classifier trained on synthetic data generated from a configurable weighted scoring rule.

The Go backend calls `POST /predict` when a shipment is created or confirmed.

## Commands

```bash
# Install dependencies (first time)
pip install -r requirements.txt

# Train the model (generates dataset + trains + saves model.pkl)
python train.py

# Run the prediction service (port 5001)
python app.py
```

The model must be trained before the service can serve predictions. `model.pkl` is generated by `train.py`.

## File structure

| File | Purpose |
|------|---------|
| `config.py` | **THE key file.** Priority factor weights and classification thresholds. Change weights here to change what matters most. |
| `dataset.py` | Generates synthetic training data using the weights from `config.py`. Produces a DataFrame of samples with labels. |
| `model.py` | `train()` function trains a RandomForestClassifier and saves to `model.pkl`. `predict()` function loads the model and returns priority + factor contributions. |
| `train.py` | One-time script: calls `dataset.py` to generate data, then `model.py` to train and save. |
| `app.py` | Flask server exposing `POST /predict`. Loads the trained model on startup. |
| `requirements.txt` | Python dependencies. |
| `model.pkl` | Serialized trained model (generated by `train.py`). |
| `CLAUDE.md` | This file — full context documentation. |

## How it works

```
config.py (weights) → dataset.py (2000 samples labeled by weights)
                    → model.py (RandomForest trained on samples)
                    → model.pkl (saved model)
                    → app.py (loads model, serves POST /predict)
```

1. `config.py` defines the priority factor weights and thresholds.
2. `dataset.py` generates random shipment samples and labels them using the weighted scoring formula from `config.py` (with 10% noise for realistic variance).
3. `model.py` trains a RandomForestClassifier on the dataset.
4. `app.py` loads the trained model and serves predictions via HTTP.

## Priority factors

| # | Factor | Weight | Description | Normalization |
|---|--------|--------|-------------|---------------|
| 1 | `shipment_type` | 3.0 | Express = highest priority | normal=0.0, express=1.0 |
| 2 | `distance_km` | 2.5 | Longer distance = more delay risk | `min(distance/2500, 1.0)` |
| 3 | `restrictions` | 2.0 | Fragile/cold chain = special handling | `count / 2.0` (0-2) |
| 4 | `time_window` | 1.5 | Morning = tighter deadline | flexible=0.0, afternoon=0.5, morning=1.0 |
| 5 | `volume_score` | 1.0 | Larger = more complex logistics | `(package_base + weight/50) / 12` capped at 1.0 |
| 6 | `route_saturation` | 0.8 | Busy route = congestion risk | used directly (0.0-1.0) |

**Scoring formula**: `total = Σ(normalized_factor × weight) / Σ(all weights)`

**Classification**:
- `score > 0.65` → **alta** (high priority)
- `score > 0.35` → **media** (medium priority)
- `score <= 0.35` → **baja** (low priority)

## How to change priorities

To change which factors matter most (e.g., make distance #1):

1. Edit `config.py`: swap weights between `distance_km` and `shipment_type`
2. Run `python train.py` (regenerates dataset + re-trains model)
3. Restart the ML service
4. Done — all new predictions use the new ordering

## Volume score calculation

The `volume_score` combines package type base size and weight:

| Package type | Base size |
|-------------|-----------|
| envelope | 1 |
| box | 5 |
| pallet | 15 |

```
volume_score = base_size + (weight_kg / 2.0)
normalized   = min(volume_score / 25.0, 1.0)
```

## Distance calculation

Distance is computed using the **Haversine formula** between province centroids.

Coordinates (lat, lng) for all 23 Argentine provinces + CABA are defined in `dataset.py`.

When the API receives a request with `origin_province` and `destination_province`, the distance in km is computed and included in the prediction.

## Synthetic dataset generation

- 2000 random samples
- Each sample has random values for all 6 factors
- Labels are assigned by the weighted scoring formula
- 10% of labels are randomly flipped to add noise (simulates real-world uncertainty)
- Stratified split ensures balanced representation of all 3 priority classes

## API

### `POST /predict`

Request:
```json
{
  "origin_province": "Córdoba",
  "destination_province": "Santa Cruz",
  "shipment_type": "express",
  "time_window": "morning",
  "package_type": "box",
  "weight_kg": 5.2,
  "is_fragile": true,
  "cold_chain": false
}
```

Response (200):
```json
{
  "priority": "alta",
  "confidence": 0.87,
  "score": 0.74,
  "factors": {
    "shipment_type":    {"value": "express",   "normalized": 1.0,  "weight": 3.0, "contribution": 0.30},
    "distance_km":      {"value": 2100,        "normalized": 0.84, "weight": 2.5, "contribution": 0.21},
    "restrictions":     {"value": 1,           "normalized": 0.5,  "weight": 2.0, "contribution": 0.10},
    "time_window":      {"value": "morning",   "normalized": 1.0,  "weight": 1.5, "contribution": 0.15},
    "volume_score":     {"value": 15.6,        "normalized": 0.31, "weight": 1.0, "contribution": 0.03},
    "route_saturation": {"value": 0.72,        "normalized": 0.72, "weight": 0.8, "contribution": 0.06}
  }
}
```

Error (503):
```json
{
  "error": "Model not loaded. Run 'python train.py' first."
}
```

### `GET /health`

Response (200):
```json
{"status": "ok"}
```

## Integration with Go backend

- The Go backend calls this service at `http://localhost:5001/predict`
- Configurable via `ML_SERVICE_URL` env var in `cmd/server/main.go`
- If the ML service is down, the Go backend logs a warning and returns the shipment without priority (empty string). No crash.
- The Go backend's `MLClient` is in `internal/service/ml_client.go`

## Adding a new factor

1. Add the factor to `config.py` with its weight
2. Add normalization logic in `dataset.py` (for training) and `model.py` (for prediction)
3. Update the Go model if the factor is a new shipment field
4. Run `python train.py` to retrain
5. Restart the service
